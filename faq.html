How do I get the bot information?

All steps use the Discord Developer Portal. For building OAuth yourself, see Discord’s OAuth2 documentation.

  1. Open the Developer Portal and open your application (or click New Application to create one).
  2. CLIENT_ID: In General Information, copy Application ID.
  3. CLIENT_SECRET: Open the OAuth2 page. Under Client information, copy Client Secret. If you do not have it, use Reset Secret and save it, you will only see it once.
  4. BOT_TOKEN: Open the Bot tab. If needed, add a bot first. Then copy the token in the Token area. If it is hidden, use Reset Token to create a new token and copy it.
  5. On the Bot tab, under Privileged Gateway Intents, turn the needed ones on.
  6. On OAuth2 → Redirects, add every exact callback URL your app uses (for example `http://localhost:3000/api/auth/discord/callback`). A typo or wrong port here causes Invalid OAuth2 redirect_uri during login.
  7. Never post Client Secret or Bot Token in public. Use environment variables, not public Git repositories or screenshots with tokens visible.

How do I share files?

Complete projects and clear notes get you hosted faster. Work through this list in order:

  1. Run the bot on yourself to fix any build and runtime errors before you open a ticket.
  2. Send a full project package: source, dependency files (package.json, requirements.txt, etc.).
  3. Share files with a normal download link. Options: Google Drive (share a folder or zip with access enabled), WeTransfer, or a ZIP you built with WinRAR or 7-Zip. One main link per ticket is easiest.
  4. In the ticket, list: language/runtime (e.g. Node 20, Python 3.12), start command.
  5. If you cannot use Google Drive, WeTransfer, or a zip file, say that at the start of the ticket and describe what you can use instead.
